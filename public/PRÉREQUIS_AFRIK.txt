# PRÉREQUIS_OFFICIELS_DU_PROJET_AFRIK
# VERSION 3.0 — Document maître (Architecture + Workflow + Identifiants)


========================================================
0. OBJECTIF DU DOCUMENT
========================================================

Ce document établit :

• l'architecture du projet Afrik  
• les identifiants uniques  
• le référentiel linguistique  
• la classification famille linguistique → langue → peuple → pays  
• les sources officielles  
• la méthodologie démographique  
• les règles de nommage  
• les fichiers maîtres CSV  
• le workflow global  
• les fichiers de suivi du workflow


========================================================
1. IDENTIFIANTS UNIQUES AFRIK
========================================================

ID pays : ISO 3166-1 alpha-3  
ID famille linguistique : FLG_xxxx  
ID langue : code ISO 639-3  
ID peuple : PPL_xxxx  


========================================================
2. RÉFÉRENTIEL LINGUISTIQUE AFRIK
========================================================

Références obligatoires :

• ISO 639-3  
• Glottolog  
• Ethnologue (SIL)  
• UNESCO  
• African Language Atlas  

Chaque langue doit indiquer :

• nom  
• code ISO  
• famille linguistique  
• sous-famille  
• nombre de locuteurs  


========================================================
3. HIÉRARCHIE AFRIK
========================================================

Niveau 1 — Famille linguistique (FLG_*)  
Niveau 2 — Langue (code ISO 639-3)  
Niveau 3 — Peuple (PPL_*)  
Niveau 4 — Pays (ISO 3166-1 alpha-3)  


========================================================
4. TERMES PÉJORATIFS / COLONIAUX
========================================================

Obligatoire :

• conserver  
• contextualiser  
• expliquer origine  
• expliquer problème  
• donner auto-appellation  
• ne jamais présenter comme neutre  


========================================================
5. MÉTHODOLOGIE DÉMOGRAPHIQUE
========================================================

Population pays = ONU / UNFPA (2025)  
% peuples par pays = CIA / SIL / UNESCO + anthropologie  
Somme = 100 %  
Chaque peuple = population totale (tous pays)  
Langue = total locuteurs  


========================================================
6. SOURCES AUTORISÉES
========================================================

ONU – UNFPA – CIA – Banque Mondiale  
Ethnologue – Glottolog – UNESCO – ALAf  
IWGIA – Vansina – Ehret – Hiernaux  
Encyclopaedia Africana  
ASCL Leiden  


========================================================
7. FICHIERS CSV MAÎTRES
========================================================

famille_linguistique.csv  
peuple_demographie_globale.csv  
pays_demographie.csv  


========================================================
8. RÈGLES DE NOMMAGE
========================================================

Fichiers TXT :  
- Familles linguistiques : FLG_xxxxx.txt  
- Peuples : PPL_xxxxx.txt (organisés par famille linguistique)  
- Pays : [code ISO 3166-1 alpha-3].txt  

ID : MAJUSCULES, sans espaces


========================================================
9. COHÉRENCE TXT ↔ CSV
========================================================

Tout TXT doit avoir une entrée correspondante dans les CSV.  
Tous les ID doivent être identiques partout.  


========================================================
10. WORKFLOW AFRIK + FICHIERS DE SUIVI
========================================================

Étapes obligatoires :

1. Familles linguistiques  
2. Langues  
3. Peuples  
4. Pays  
5. Démographies CSV  
6. Validation  
7. Publication  

FICHIERS DE SUIVI OBLIGATOIRES :

1. WORKFLOW_AFRIK_STATUS.md  
   → checklist humaine  

2. workflow_status.csv  
   → suivi machine  
   → statuts : pending / in_progress / done  

Aucune étape du workflow ne peut être considérée comme terminée  
si elle n’est pas reflétée dans ces deux fichiers.
