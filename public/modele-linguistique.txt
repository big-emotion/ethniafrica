## DIRECTIVES DE RÉDACTION (À RESPECTER STRICTEMENT)

Ces directives garantissent un parsing fiable et une cohérence entre toutes les fiches.

### 1. Formatage du texte
- ❌ **PAS de symboles** : ~, >, <, ±, *, etc.
- ❌ **PAS de markdown** : **gras**, *italique*, `code`, etc.
- ✅ **Texte brut uniquement**
- ✅ **Utiliser des mots** : "environ", "plus de", "moins de"

### 2. Approximations numériques (locuteurs, langues)
**Format standardisé** :
- "environ X millions" | "plus de X millions" | "moins de X millions"
- "environ X" | "plus de X" | "moins de X"

**Exemples corrects** :
- "environ 500 millions"
- "plus de 30 millions"
- "moins de 10 000"
- "environ 375 langues"

**Exemples incorrects** :
- "~500 millions" ❌ (symbole tilde)
- ">30 millions" ❌ (symbole supérieur)
- "<10 000" ❌ (symbole inférieur)
- "±500" ❌ (symbole plus-moins)

### 3. Aires géographiques (Header + Section 1)
**Format** : Liste séparée par virgules, SANS tirets initiaux

**Exemple correct** :
```
Aire géographique : Afrique du Nord, Corne de l'Afrique, Sahel et Sahara, Extensions sahéliennes
```

**Exemples incorrects** :
```
Aire géographique :
  - Afrique du Nord
  - Corne de l'Afrique
  - Sahel et Sahara
```
❌ (tirets, format liste)

```
Aire géographique : - Afrique du Nord - Corne de l'Afrique
```
❌ (tirets parasites)

### 4. Peuples associés (Section 2)
**Format strict** : Lister 5 à 10 EXEMPLES représentatifs (pas exhaustif)
- Format : "- Peuple X : Nom (PPL_ID)"
- ❌ **PAS de raccourci "(autres : ...)"**

**Exemples corrects** :
```
# 2. Peuples associés
- Peuple 1 : Kongo (PPL_KONGO)
- Peuple 2 : Luba (PPL_LUBA)
- Peuple 3 : Zoulou (PPL_ZULU)
- Peuple 4 : Shona (PPL_SHONA)
- Peuple 5 : Tswana (PPL_TSWANA)
```

**Exemple incorrect** :
```
# 2. Peuples associés
- Peuple 1 : Kongo (PPL_KONGO)
- Peuple 2 : Luba (PPL_LUBA)
- (autres : Shona, Tswana, Ganda...)
```
❌ (pattern "autres" interdit)

### 5. Longueur cible
- **Header décolonial** : 15-20 lignes
- **Chaque section** : 15-30 lignes
- **Total cible** : 100-150 lignes maximum
- ⚠️ **Éviter les fiches > 200 lignes** (trop verbeuses)

### 6. Caractéristiques linguistiques (Section 3)
- Descriptions factuelles avec exemples concrets
- Pas de listes à puces excessives avec tirets
- Privilégier les phrases complètes
- Pas de formatage markdown
